Git Intermediário

Aprenda o nível intermediário de Git.

Introdução

Nesta etapa, você já entende os comandos básicos do Git e está pronto para trabalhar com situações mais complexas do dia a dia, como gerenciamento de branches, resolução de conflitos e colaboração com várias pessoas no mesmo repositório. O objetivo desta página é ensinar conceitos intermediários que tornam o seu fluxo de trabalho mais organizado e eficiente.

Trabalhando com Branches

Branches permitem criar versões paralelas do seu projeto, evitando bagunça e facilitando novos recursos sem quebrar o código principal.

O que você aprenderá:

  • Criar branches para novas funcionalidades
  • Alternar entre branches
  • Entender a diferença entre main, develop e branches de features
  • Remover branches que não são mais necessárias

Comandos importantes:

git branch nome-da-branch

git checkout nome-da-branch

git switch nome-da-branch

git branch -d nome-da-branch

Resolvendo Conflitos

Conflitos acontecem quando duas pessoas alteram a mesma parte do arquivo. Eles não são erros: são um processo natural em colaboração

Você verá:

  • Como identificar conflitos
  • Como resolver manualmente
  • Como confirmar a resolução

Mensagem típica de conflito:

<<< HEAD Seu código ======= Código de outra pessoa >>> branch-remota

Git Stash - Guardando trabalho temporariamente

Ótimo para quando você precisa trocar de tarefa rapidamente sem perder mudanças em andamento.

Comandos úteis:

git stash

git stash list

git stash apply
