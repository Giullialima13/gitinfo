Git Avançado

Aprenda o nivel Avançado de Git.

Introdução

Nesta página você explorará funcionalidades que fazem o Git ser tão poderoso. Essas técnicas são usadas em grandes empresas, projetos open-source e fluxos de trabalho complexos.

Rebase - História limpa e organizada

Branches permitem criar versões paralelas do seu projeto, evitando bagunça e facilitando novos recursos sem quebrar o código principal.

O que você aprenderá:

  • Quando usar rebase em vez de merge
  • Perigos e boas práticas
  • Como realizar um rebase interativo

Exemplo:

git rebase -i HEAD~5

Comandos importantes:

git branch nome-da-branch

git checkout nome-da-branch

git switch nome-da-branch

git branch -d nome-da-branch

Cherry-Pick - Selecionando commits específicos

Permite aplicar um commit isolado de outra branch.

git cherry-pick <hash>

Ótimo para quando:

  • Você precisa trazer uma correção urgente
  • Quer testar apenas uma mudança específica

Tags — Marcando versões

Tags identificam pontos importantes no desenvolvimento, como versões estáveis.

git tag -a v1.0 -m "Primeira versão estável"

git push --tags

Git Bisect - Encontrando bugs automaticamente

Um dos comandos mais poderosos do Git. Permite descobrir qual commit introduziu um bug através de busca binária.

Exemplo:

git bisect start

git bisect bad

git bisect good <hash>

Hooks - Automatizando tarefas

Hooks permitem executar scripts automaticamente quando certas ações acontecem.

Exemplos:

  • Impedir commits com erros
  • Rodar testes antes de enviar ao repositório
  • Realizar formatação automática de código

Arquivos ficam em:

.git/hooks/

Submodules - Projetos dentro de projetos

Para quando você precisa incorporar um repositório Git dentro de outro.

sql:

git submodule add <URL>

git submodule update --init

Fluxo Avançado Profissional

Fluxo de Trabalho Avançado(Exemplo)

  1. main → versão estável
  2. develop → código em desenvolvimento
  3. feature/* → novas funcionalidades
  4. release/* → estabilizar versões
  5. hotfix/* → correções urgentes
